# 玉米生产力预测平台（公开版）

## 项目介绍

这是一个面向多类玉米算法服务的 Web 平台公开版仓库，保留了我负责的前后端平台代码，包括统一入口、参数配置、任务调度、结果预览、地图展示和基础数据管理等能力。

由于项目中的部分玉米算法源码、模型文件和业务数据由合作方提供，涉及保密要求，因此本仓库仅公开平台层、服务接入层和交互层代码，不包含私有算法核心实现。

## 技术栈

- 前端：Vue 3、Vite、Element Plus、Pinia、MapTalks
- 后端：FastAPI、Tortoise ORM、MySQL
- 其他能力：异步任务管理、文件上传下载、GeoJSON/WMS 地图展示、环境变量配置

## 核心能力

- 多算法统一平台入口：支持算法选择、参数配置、任务提交、任务历史查看和结果下载
- 任务调度与状态跟踪：围绕 `task_id` 实现任务创建、进度查询、取消、删除、清理和历史记录管理
- 参数配置与持久化：支持多类算法参数读取、更新、重置和说明展示
- 结果预览与空间化展示：支持结果文件预览、统计信息展示、GeoJSON 地图渲染和 WMS 图层加载
- 基础数据管理：支持分类文件管理、JSON/表格上传、在线表格编辑、列名修改、删列、整表保存和导出
- 配置环境化：数据库、地图服务、外部算法接口和上传地址均改为环境变量或代理配置

## 公开版保留内容

- `corn-system`：前端页面与交互逻辑
- `app-py`：后端网关、任务编排、数据管理和接口层代码
- 项目说明文档、环境变量示例文件、公开版 Git 忽略规则

## 未开源内容

以下内容已从 Git 跟踪中排除，不应上传到公开仓库：

- 合作方或第三方提供的玉米算法源码
- 编译后的算法程序、模型运行时、训练产物和相关私有工具
- 样例输入数据、气象数据、矢量栅格数据、输出结果文件
- 本地日志、缓存、临时文件和原始前端仓库 Git 历史

## 本地运行

### 前端

1. 将 `corn-system/.env.example` 复制为 `corn-system/.env`
2. 在 `corn-system` 目录安装依赖
3. 按本地服务地址配置环境变量后启动前端

### 后端

1. 将 `app-py/.env.example` 复制为 `app-py/.env`
2. 在 `app-py` 目录安装依赖
3. 配置数据库和算法服务地址后启动 FastAPI 服务

## 说明

- 公开仓库重点展示的是平台化、工程化和前后端协同能力，不代表完整生产环境部署内容
- 某些算法页面依赖私有服务或私有模型资源，公开版中保留了调用入口和平台流程，但不包含保密实现
- 如需新建 Git 仓库或上传 GitHub，请务必在当前项目根目录操作，以确保顶层 `.gitignore` 生效
